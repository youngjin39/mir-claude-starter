/skill-creator 이 레포의 모든 세션에 대해 컨텍스트와 토큰 효율성 시각화를 보여주는 스킬을 만들어줘. 점수를 매기고, 어떤 가이드라인과 기준으로 채점했는지도 함께 보여줘. 그리고 비용($) 관점에서 효율을 어떻게 개선할 수 있는지 개선안도 공유해줘.

각 시나리오별로 비효율을 탐지하는 python 스크립트를 만들어줘:
탐지기
1. 컨텍스트 창이 무한정 부풀어 오르는 경우 — 컨텍스트가 100k 토큰을 넘는 상태가 20+ turn 이어지면서 한 번도 50% 이상 감소(compact 가 호출됐다는 신호)하지 않았는지 검사. 낭비 = turn 별로 80k 를 초과한 토큰 합계.
2. 거대한 도구 출력이 컨텍스트를 부풀리는 경우 — tool_result 가 50k chars(약 12.5k 토큰) 이상이면 플래그. 낭비는 남은 turn 수만큼 곱해진다 (한 번 들어가면 컨텍스트에 계속 남으므로).
3. 프롬프트 캐싱 활용이 저조한 경우 — 한 turn 의 cache hit ratio 가 50% 미만이면서 컨텍스트가 30k 토큰을 넘으면 플래그. 낭비 = cache_create 토큰 × (cache create 단가 − cache read 단가). 즉, cache hit 대비 더 비싸게 지불한 만큼.
4. 완전히 동일한 도구 호출이 중복되는 경우 — 각 (tool name, input) 쌍을 SHA-256 으로 해싱. 같은 해시가 다시 나타나면 그 결과 전체가 낭비.
5. Task / 서브에이전트 위임 과다 — Task 호출이 5회 이상이면서 서브에이전트당 평균 turn 수가 ≤ 3 (= 사소한 작업을 비싼 서브에이전트로 떠넘겼다는 의미) 일 때 플래그. 낭비 = 초과 호출당 추정 오버헤드 토큰.
